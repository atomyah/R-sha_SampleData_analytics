## rates.yamlについて

- **rates.yaml**は全国対応。R社サンプルデータ.xlsxの「都道府県」列から県名を読んでスタッフの所属県に応じて自動で正しい健保率を引く設計。rates.yamlを全47都道府県版に作り直してある。

- スタッフごとに所属県が違っても自動で正しいけんぽ料率を引けるようになった。万一、都道府県が空欄の契約があった場合は default_prefecture（既定:神奈川県）にフォールバックする。

- ひとつ実務上の注意：健康保険料は「スタッフの居住地」ではなく「加入している適用事業所（＝雇用主である派遣元の事業所）の所在地」の料率で決まる。今回は契約データの都道府県をそのまま使う作りにしてるが、もし適用事業所が特定の県（たとえば本社のある県）に集約されているなら、本来はその県の料率で全員計算するのが正しい、というケースもあり得る。**システムが「契約の都道府県」で引いているのか「適用事業所の県」で引いているのかは確認必要**。

- 来年度の料率改定時は、rates.yamlの **health: ブロック**の数字を令和8年度の値に置き換えるだけでOK。Claude Codeに「協会けんぽ令和8年度の全都道府県料率にrates.yamlを更新して」と頼めばよい。
