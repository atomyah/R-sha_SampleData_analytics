# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## リポジトリの目的

HRTech SaaSからエクスポートしたExcelサンプルデータを検証する単一スクリプト（`src/サンプルデータ検証_yaml版.py`）。契約シートと勤怠シートから請求額・支払額を再計算し、「サンプルデータ概要」シートの目標値と突き合わせて、ケースごとに OK / NG を出力する。NG時はまず社保フラグと有給行を疑う。

料率・税額表・各種設定は `src/rates.yaml` に外出しされており、改定時は原則 YAML だけを編集する（コードは触らない）。

## レイアウト

```
src/
  サンプルデータ検証_yaml版.py   # 本体スクリプト
  rates.yaml                     # 料率・税額表・設定
  R社サンプルデータ.xlsx         # 検証対象サンプル
README.md                        # rates.yaml の運用メモ
```

## 実行方法

```bash
# 依存（requirements.txt はないので個別にインストール）
pip install openpyxl pyyaml --break-system-packages

# 基本実行
python src/サンプルデータ検証_yaml版.py src/R社サンプルデータ.xlsx --rates src/rates.yaml

# よく使うフラグ
python src/サンプルデータ検証_yaml版.py src/R社サンプルデータ.xlsx \
    --rates src/rates.yaml --tax-year 2026 --verbose
```

`--round-unit` / `--pay-tol` は未指定時 `rates.yaml` の `settings` ブロックの値を使う。CLI で渡せば上書きできる。

全ケースが通れば終了コード 0、1件でも NG なら 1 を返す。`--pay-tol` を設けてあるのは、源泉所得税が税額表側で丸め込まれて数十円ずれ得るため。請求側は完全一致（差0円）が必須。

## アーキテクチャ

処理は全て1ファイル内に収まっている。実行順で主要部品を並べると以下の通り。

1. **`load_config(path)`** — `rates.yaml` を読み、グローバルの `RATES` / `SETTINGS` / `GENSEN` を構築する。`insurance.active_year` のキー（例: `reiwa7`）でその年度のサブブロックを引き、社会保険料率・消費税率・全47都道府県の健保率を `RATES` にまとめる。`withholding` 下の各年度の表は `_make_gensen` でクロージャ化され、`GENSEN[年]` で呼び出せる関数になる。
2. **`_make_gensen(table)`** — 月額表 甲欄 電算機計算の特例を、`salary_deduction` / `dependent_deduction` / `basic_deduction` / `tax_brackets` のテーブル4枚から組み立てるクロージャ。`[上限, kind/rate, 率, 加算]` 形式の表を上から舐め、最初に該当した枠で計算する。戻り値は10円未満四捨五入。
3. **`find_sheets(wb)`** — シート名の先頭がケース記号（①②③…）になっているものを抽出し、契約／勤怠シートをケース単位でペアリングする。マル数字プレフィックスのないシートは無視される。
4. **`parse_overview(wb)`** — 「概要」シートを読む。C列がケース別の請求目標額、D列がスタッフ別の支払目標額リスト（順不同。後段で最近接マッチング）。
5. **`split_attendance(contract, recs, round_unit, legal_daily)`** — 打刻データを時間バケットに振り分ける。実働分 = (終了−開始) − 休憩 − 深夜休憩 を `round_unit` で切捨て。バケットは 所定内（`base`）／ 法定内残業（`nai`、所定 < `legal_daily` のときだけ発生）／ 法定外残業（`gai`、`legal_daily` 超）。打刻が空で有給フラグまたは `支払い料金`／`請求料金` の格納値がある行は有給日として処理し、格納値をそのまま支払に流す（**有給は請求対象外**）。
6. **`calc_contract(...)`** — 契約ごとの結果を組み立てる:
   - 請求税抜 = Σ(時間 × 請求単価)。税込は後段で ×(1 + `RATES["shohi"]`)。本サンプルは全額10%課税・交通費請求なし前提。
   - 本給 = Σ(時間 × 支払単価) + 有給日の支払い料金。
   - **健保率は契約データの「都道府県」列で選択**。`RATES["health"]` から引き、ヒットしなければ末尾の「都/道/府/県」を除去して再探索、それでも無ければ `default_prefecture`（YAML 既定: 神奈川県）にフォールバック。
   - 健保 = 標準報酬 × (健保率 + 介護加入なら介護率) / 2。厚年 = 標準報酬 × 厚年率 / 2。雇用 = (本給 + 交通費) × 雇用率。それぞれ対応する加入フラグでゲートされる。
   - 差引支給額 = 本給 − 社保 + 交通費 − 源泉 − 住民税。住民税は本サンプルではハードコード 0。
7. **`main()`** — ケース単位で契約合算→税込→請求判定。続いてスタッフ別の差引支給額を、概要シートの支払目標リストに対し最近接の貪欲マッチングで対応付ける。集合は一致するが順序が違う場合（表示順入替の疑い）には警告を出す。

## 料率・設定の編集ポイント

すべて `src/rates.yaml` で完結する設計:

| 編集対象 | YAML 内の場所 |
|----------|---------------|
| 適用年度の切替 | `insurance.active_year`（値を新年度のサブブロック名に変更） |
| 健保率（都道府県別）/ 厚年 / 雇用 / 介護 / 消費税 | `insurance.<年度キー>.{health,pension,employment,care,consumption}` |
| 都道府県不明時のフォールバック | `insurance.default_prefecture` |
| 切捨て単位・支払許容差・法定労働時間 | `settings.{round_unit_min,pay_tolerance_yen,legal_daily_hours}` |
| 源泉所得税の月額表 | `withholding.<年>.{salary_deduction,dependent_deduction,basic_deduction,tax_brackets}` |

新年度に切り替える手順:
1. `rates.yaml` の `insurance:` 配下に新しい年度サブブロック（例 `reiwa8:`）を追加。
2. `insurance.active_year` の値を新ブロック名に変更。
3. （税額表が改定されたなら）`withholding:` に新しい年度キーを追加し、CLI で `--tax-year` を新年度に。

`サンプルデータ検証_yaml版.py` の編集は不要。

## 健保率「契約の都道府県 vs 適用事業所の都道府県」

本ツールは契約データの「都道府県」列をそのまま使う実装。本来、健康保険料率は **加入している適用事業所（雇用主である派遣元事業所）の所在地** の料率で決まるため、適用事業所が特定の県に集約されている運用ではズレが生じ得る。詳細は `README.md` を参照。

## スクリプトが依存するシート規約

- 契約シートは名前に「契約」、勤怠シートは「勤怠」、概要シートは「概要」を含み、それぞれ先頭にケース記号（①②…）が付く前提。
- 契約と勤怠は `契約番号` 列で結合する。
- 勤怠シートで参照する列: `開始日時`, `終了日時`, `休憩時間（分）`, `深夜休憩時間（分）`, `1日有給休暇フラグ`, `半日有給休暇フラグ`, `支払い料金`, `請求料金`, `通勤交通費：支給`。
- 契約シートで参照する列: `所定労働時間`, `都道府県`, `請求料金 基本単価` / `法定内残業単価` / `法定外残業単価`, `支払料金 基本単価` / `法定内残業単価` / `法定外残業単価`, `標準報酬月額`, `健康保険加入`, `厚生年金保険加入`, `介護保険加入`, `雇用保険加入`, `扶養人数`, `料金名`。

これらをリネームするとスクリプトは沈黙したまま壊れる（`num()` がキー欠落時に 0 を返すため、エラーにならず誤った計算結果が出る）。
